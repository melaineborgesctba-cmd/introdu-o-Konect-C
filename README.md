2. Concepção
a. Descrição da visão geral do projeto 
O projeto KONECT-C propõe a criação de uma Interface de Conexão Digital voltada a promover o comércio e os serviços locais, funcionando como um Marketplace Comunitário. A interface será desenvolvida utilizando HTML e CSS para garantir acessibilidade e leveza, com foco em design responsivo. O nome KONECT-C (lido como "Conecte-se") utiliza a letra "C" para reforçar a ideia de Conexão entre Comunidade e Comércio, ao mesmo tempo que homenageia a área de programação. O objetivo é oferecer uma plataforma visualmente intuitiva onde moradores possam encontrar e interagir facilmente com negócios e profissionais da região.

b. Objetivo do Projeto
Objetivo Principal: Desenvolver uma interface front-end acessível, usando HTML e CSS (e potencialmente um ambiente de desenvolvimento como Visual Studio ou outro editor), capaz de exibir e organizar dados de comércio e serviços locais de forma dinâmica e intuitiva.

Objetivo de Negócio: Criar um canal digital eficiente para o Marketplace local, aumentando a visibilidade dos pequenos negócios e fortalecendo a economia da comunidade.

c. Escopo do Produto
• Descrição do Produto:
O KONECT-C é uma Interface Web (Landing Page/Site) que atua como um diretório digital. Ele apresenta informações de negócios e profissionais locais, permitindo aos usuários:

Visualizar e Pesquisar: Encontrar comércios por categoria (ex: gastronomia, serviços, saúde).

Informações Detalhadas: Acessar dados de contato, horário de funcionamento e localização dos estabelecimentos.

Design Intuitivo: Apresentar uma experiência de usuário (UX/UI) clara, seguindo o guia de estilo com estética de "ferramentas de ofício" (presente no logo).

Principais Entregas
MVP (Produto Mínimo Viável):

Estrutura HTML/CSS completa do layout da página principal.

Exibição estática de, no mínimo, 5 estabelecimentos fictícios com seus dados básicos.

Design responsivo inicial.

Incremento 1:

Implementação de filtros e barra de pesquisa funcional (simulada ou básica, dependendo do uso de JavaScript).

Refinamento do CSS para atingir a identidade visual completa e a acessibilidade.

Incremento 2:

Criação de uma seção ou página separada focada na captação de novos negócios para o Marketplace (Landing Page de Marketing).

Entrega Final:

Código-fonte limpo, bem estruturado e publicado no GitHub.

Documentação completa (incluindo todas as seções do PI) e apresentação final.

• Critérios de Aceite:
Critérios Qualitativos (Validação do Produto)
Identidade Visual Consistente: O design da interface (HTML/CSS) deve aderir fielmente ao Guia de Estilo, utilizando a paleta de cores Steampunk/Cobre (#B87333) de forma estratégica, especialmente em elementos interativos.

Usabilidade e Navegação: A navegação entre as categorias e o resultado da busca deve ser intuitiva e exigir o mínimo de cliques para o usuário acessar a informação de contato de um comércio.

Design Responsivo: O layout deve se adaptar e ser plenamente funcional em, no mínimo, três resoluções diferentes (Desktop, Tablet e Mobile), sem barras de rolagem horizontais ou elementos sobrepostos.

Organização do Código: O código HTML e CSS deve ser limpo, bem indentado, comentado de forma clara e seguir as melhores práticas do W3C.

Critérios Quantificáveis (Determinam o Sucesso)
Escopo Mínimo: O MVP deve exibir corretamente, pelo menos, 5 estabelecimentos com todas as informações básicas (nome, categoria, contato).

Validação de Código: O código HTML deve passar na validação do W3C (ou similar) com, no máximo, 3 erros não críticos.

Tempo de Carregamento: A página principal (com os 5 estabelecimentos) deve carregar completamente em menos de 3 segundos em uma rede padrão.

Taxa de Conclusão do Teste: Pelo menos 80% dos usuários no teste de usabilidade (protótipo Quant-UX) devem conseguir localizar um estabelecimento específico e o seu telefone de contato com sucesso.

Conclusão do Incremento 1: O filtro/barra de pesquisa deve estar totalmente implementado e funcional para 100% das categorias definidas.

4. Desenvolvimento
   a. Processo de Software
Para a gestão e desenvolvimento da interface web, a metodologia Kanban será utilizada.

Metodologia Escolhida: Kanban.

Justificativa: O Kanban se adapta bem a projetos com escopo de entrega visual (como uma interface Front-end) e permite que a equipe se concentre no fluxo de trabalho contínuo, priorizando as tarefas em um Quadro Visual (Trello) e mantendo o foco no limite de trabalho em progresso (WIP).

Ciclo de Entrega: O desenvolvimento será dividido em Incrementos (conforme Seção 2.c), com foco em finalizar e revisar a funcionalidade completa de cada incremento antes de iniciar o próximo.

d. Instruções para Download e Execução
Como o projeto é uma interface Front-end (HTML/CSS), a execução é extremamente simples, o que facilita o acesso para o usuário final e o avaliador.

Clonar o Repositório: git clone <link do projeto>

Navegar até o Diretório do Projeto: cd konect-c

Executar a Aplicação:

Simplesmente localize o arquivo index.html no diretório.

Clique duas vezes no arquivo index.html ou abra-o com qualquer navegador (Chrome, Firefox, Edge, etc.). O navegador carregará a interface web do KONECT-C
